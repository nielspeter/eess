# ADR-999: Non-vacuity probe (invalid tier)

NON-VACUITY FIXTURE — this ADR declares a proper `## Enforcement` table (so the
`adr/enforcement-declared` check passes) but its single clause claims **tier 9**,
which is outside the valid range 1–5. The `adr/valid-tiers` check in the
`adrEnforcement` preset MUST therefore report a violation and throw. If it did
not, the tier gate would be vacuous. See scripts/nonvacuity/bad-adr.mjs.

The mechanism cites no files and no `it()` titles on purpose, so the
`adr/citations-resolve` check stays green — isolating the tier failure.
